# Confluence Mermaid Diagrams

Insert native Mermaid diagrams into Confluence Cloud pages using the "Mermaid Diagrams for Confluence" plugin (extensionKey: `mermaid-cloud`).

## The Problem

Confluence Cloud does not render Mermaid natively. Markdown code blocks with ` ```mermaid ` are stored as plain code blocks, not rendered diagrams. The Atlassian MCP tool does not support inserting extension macros via markdown format.

## How the Plugin Works

The "Mermaid Diagrams for Confluence" plugin stores diagrams as:

1. **A text attachment** (the mermaid markup) — filename without extension, `text/plain`
2. **A PNG attachment** (preview image) — `{filename}.png`, auto-generated by the plugin on first render
3. **An ADF extension node** referencing the filename

The plugin renders the diagram as SVG on page view. The PNG is a fallback for export/preview.

## Step-by-Step Workflow

### Step 1: Prepare the Mermaid markup

Write standard Mermaid syntax. Supported types: `erDiagram`, `graph`, `flowchart`, `sequenceDiagram`, `stateDiagram-v2`, `gantt`, `classDiagram`, `gitGraph`, etc.

### Step 2: Upload markup as text attachment

```bash
AUTH=$(echo -n "email:api-token" | base64)
PAGE_ID=<pageId>

# Write markup to temp file
cat > /tmp/mermaid_diagram.txt << 'EOF'
graph TD
    A[Start] --> B[End]
EOF

# Upload as text/plain attachment
curl -s -X POST \
  -H "Authorization: Basic $AUTH" \
  -H "X-Atlassian-Token: nocheck" \
  -F "file=@/tmp/mermaid_diagram.txt;filename=my-diagram;type=text/plain" \
  "https://{site}.atlassian.net/wiki/rest/api/content/${PAGE_ID}/child/attachment"
```

The filename (without extension) becomes the diagram identifier. Use kebab-case: `current-schema`, `call-flow`, etc.

### Step 3: Build ADF with the mermaid-cloud extension node

Each diagram is an `extension` node in ADF:

```json
{
  "type": "extension",
  "attrs": {
    "layout": "default",
    "extensionType": "com.atlassian.confluence.macro.core",
    "extensionKey": "mermaid-cloud",
    "parameters": {
      "macroParams": {
        "filename": {
          "value": "my-diagram"
        },
        "revision": {
          "value": "1"
        }
      },
      "macroMetadata": {
        "macroId": {
          "value": "<random-uuid>"
        },
        "schemaVersion": {
          "value": "1"
        },
        "placeholder": [
          {
            "type": "icon",
            "data": {
              "url": "https://mermaid.stratus-addons.com/images/mermaid144.png"
            }
          }
        ],
        "title": "Mermaid Diagrams for Confluence"
      }
    },
    "localId": "<random-uuid>"
  }
}
```

Key fields:
- `filename.value` — matches the attachment filename (no extension)
- `revision.value` — increment when updating the markup
- `macroId.value` — unique UUID per macro instance
- `localId` — unique UUID per node

### Step 4: Upload the full ADF page

```bash
# Build request with ADF body
python3 -c "
import json
with open('/tmp/adf.json') as f:
    adf = json.load(f)
request = {
    'id': '<pageId>',
    'status': 'current',
    'title': 'Page Title',
    'body': {
        'representation': 'atlas_doc_format',
        'value': json.dumps(adf)
    },
    'version': {
        'number': CURRENT_VERSION + 1,
        'message': 'Add Mermaid diagrams'
    }
}
with open('/tmp/request.json', 'w') as f:
    json.dump(request, f)
"

# Upload
curl -s -w "%{http_code}" \
  -X PUT \
  -H "Authorization: Basic $AUTH" \
  -H "Content-Type: application/json" \
  -d @/tmp/request.json \
  "https://{site}.atlassian.net/wiki/api/v2/pages/<pageId>"
```

## Python Helper for Building ADF

```python
import uuid, json

def mermaid_macro(filename):
    """Create an ADF extension node for a Mermaid diagram."""
    return {
        "type": "extension",
        "attrs": {
            "layout": "default",
            "extensionType": "com.atlassian.confluence.macro.core",
            "extensionKey": "mermaid-cloud",
            "parameters": {
                "macroParams": {
                    "filename": {"value": filename},
                    "revision": {"value": "1"}
                },
                "macroMetadata": {
                    "macroId": {"value": str(uuid.uuid4())},
                    "schemaVersion": {"value": "1"},
                    "placeholder": [{"type": "icon", "data": {"url": "https://mermaid.stratus-addons.com/images/mermaid144.png"}}],
                    "title": "Mermaid Diagrams for Confluence"
                }
            },
            "localId": str(uuid.uuid4())
        }
    }

def heading(level, text):
    return {"type": "heading", "attrs": {"level": level}, "content": [{"type": "text", "text": text}]}

def para(text):
    return {"type": "paragraph", "content": [{"type": "text", "text": text}]}

def ruler():
    return {"type": "rule"}

# Build a page with diagrams
adf = {
    "version": 1,
    "type": "doc",
    "content": [
        heading(2, "My Diagram"),
        mermaid_macro("my-diagram"),
        para("Description of the diagram."),
        ruler(),
    ]
}
```

## Updating an Existing Diagram

1. Upload new markup with the same filename — Confluence overwrites the attachment
2. Increment `revision.value` in the macro parameters
3. Update the page ADF

## Multiple Diagrams on One Page

Each diagram needs a unique filename. Upload all markups first, then build the ADF with all extension nodes.

## Rules

1. **Filename must match** between the attachment and the `macroParams.filename.value`
2. **Each macro needs unique UUIDs** for `macroId` and `localId`
3. **The PNG preview is auto-generated** by the plugin on first render — you only need to upload the text markup
4. **Use `text/plain`** content type for the markup attachment
5. **Use REST API v2** (`/wiki/api/v2/pages/`) for the page update with ADF body
6. **Increment revision** when updating markup to force re-render

## Auth

Same as confluence-edit skill: Basic Auth with `email:api-token`.

API tokens: https://id.atlassian.com/manage-profile/security/api-tokens

## Prerequisites

The "Mermaid Diagrams for Confluence" plugin must be installed on the Confluence instance. Check by searching for "mermaid" in the editor's macro menu (+ button → search "mer").
